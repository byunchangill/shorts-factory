# AI Shorts Factory — Web UI/UX 설계 기획서

> 터미널 기반 파이프라인을 웹 UI로 전환하기 위한 UI/UX 설계 기획서
>
> Version 1.0 | 2026.03.12

---

## 1. 프로젝트 개요

### 1.1 배경

AI Shorts Factory는 YouTube 인기 영상을 분석하고, 구조만 오마주하여 저작권 안전한 새 콘텐츠를 자동 생산하는 반자동 파이프라인입니다. 현재 터미널(CLI) 기반으로 작동하며, 비개발자도 쉽게 사용할 수 있는 웹 UI가 필요합니다.

### 1.2 현황 분석

| 구분 | 현재 (CLI) | 목표 (Web UI) |
|------|-----------|--------------|
| **사용 방법** | 터미널 명령어 입력 | 웹 브라우저에서 클릭/선택 |
| **사용자 인터랙션** | 텍스트 프롬프트 응답 | 버튼, 카드, 프리뷰 UI |
| **상태 확인** | JSON 파일 직접 확인 | 실시간 상태 대시보드 |
| **에셋 프리뷰** | 파일 탐색기로 개별 확인 | 인라인 프리뷰 + 버전 비교 |
| **재생성** | 명령어 + 씨 변경 입력 | 원클릭 재생성 버튼 |

### 1.3 3가지 콘텐츠 타입

| 타입 | 키워드 | 핵심 | 영상화 | 스타일 |
|------|--------|------|--------|--------|
| 🐶 **강아지** | 강아지, 개, 반려견 | 감정선, 캐릭터 일관성 | ✅ 필요 | 픽사 3D CGI |
| 📖 **썰툰** | 썰, 썰툰, 실화 | 훅, 반전, 속도감 | ❌ 이미지만 | 한국 웹툰 |
| 💊 **건강** | 건강, 효능, 상식 | 신뢰감, 가독성 | ✅ 필요 | 플랫 인포그래픽 |

## 2. UI/UX 설계 목표

### 2.1 핵심 설계 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **스텝 중심 탐색** | 7단계 워크플로의 현재 위치를 항상 시각적으로 표시하고, 다음 액션을 명확히 제시 |
| 2 | **인라인 프리뷰** | 이미지, 영상, 대본 등 모든 에셋을 화면 내에서 즉시 확인 가능 |
| 3 | **원클릭 액션** | 승인, 재생성, 버전 복원 등 핵심 액션을 버튼 한 번으로 실행 |
| 4 | **실시간 상태 추적** | 상태 머신 기반으로 진행률, 단계, 에러를 대시보드에 실시간 반영 |
| 5 | **비파괴적 수정** | 에셋을 개별 선택하여 수정하고, 나머지는 보존. 전체 재생성 방지 |

### 2.2 타겟 사용자

**주 사용자:** 1인 콘텐츠 크리에이터. 기술적 배경 없이 키워드 입력부터 최종 다운로드까지 웹에서 완결.

**사용 환경:** 데스크톱 브라우저 (Chrome 기준, 최소 1280px)

| 영역 | 역할 |
|------|------|
| **왼쪽 사이드바 (240px)** | GNB 메뉴 + 진행 중 Job 목록 + 7단계 스텝 네비게이션 (Job 상세 진입 시) |
| **메인 콘텐츠** | 현재 단계의 주요 작업 영역. 카드 그리드, 편집기, 프리뷰 등 표시 |
| **우측 패널 (320px)** | 상태 요약, 로그, 실시간 알림. 접기/펼치기 가능 |

## 4. 화면 설계

### 4.1 대시보드 (`/dashboard`)

**목적:** 전체 Job 목록을 한눈에 파악하고, 새 작업을 빠르게 시작할 수 있는 진입점.

**상단 요약 카드 4개:** 전체 Job 수 | 오늘 생성 | 승인 대기 | 에러 발생. 하단에 Job 목록 테이블 (ID, 타입, 키워드, 단계, 상태, 액션).

### 4.2 새 작업 생성 (`/jobs/new`)

**목적:** 키워드와 콘텐츠 타입을 선택하여 새 Job 생성.

**UI 구성:** 키워드 입력 필드 + 3개 콘텐츠 타입 카드 (🐶/📖/💊) + 자동 타입 감지 + "리서치 시작" CTA 버튼.

### 4.3 YouTube 리서치 & 선택 (`/jobs/:id/research`)

**목적:** TOP 5 영상을 분석 결과와 함께 카드로 제시. 사용자가 1개 선택.

**카드 구성:** 썸네일 + 제목 + 지표(조회/좋아요/댓글) + 5차원 점수 바 차트 + 분석 요약 + "이 영상으로 오마주" 버튼.

### 4.4 대본 편집/승인 (`/jobs/:id/script`)

**목적:** 생성된 대본을 시각적으로 확인하고 수정/승인.

**화면 구성:** 왼쪽 씬 타임라인 (나레이션+비주얼+감정/카메라) + 오른쪽 편집기 + 하단 유사도 게이지 3개 + 건강 컴플라이언스 패널 + 메타정보(제목 후보/해시태그) + 승인/수정/재생성 버튼.

### 4.5 이미지 생성 & 관리 (`/jobs/:id/images`)

**목적:** 씬별 이미지를 그리드로 확인하고, 개별 재생성/버전 복원/승인.

**화면 구성:** 상단 Style Bible + Character Bible 카드 + 메인 씬 이미지 그리드 (씬번호+프리뷰+상태배지+호버 나레이션) + 개별 재생성/복원/승인 + 전체 승인 버튼.

### 4.6 영상 생성 & TTS 합성 (`/jobs/:id/video`)

**목적:** 영상화 + TTS + 자막/BGM 합성 처리 및 프리뷰.

**화면 구성:** 씬별 영상 클립 그리드 + 전체 플레이어 + 모션 프리셋 드롭다운 + TTS 음성/속도 설정 + 타이밍 싱크 시각화 + 합성 버튼.

### 4.7 QC 검수 & 패키징 (`/jobs/:id/qc`, `/jobs/:id/export`)

**목적:** 6개 항목 QC 결과 확인 후 최종 패키징.

**QC 6개 항목:** 저작권 안전성, 이미지 일관성, 영상 규격, 음성 싱크, 콘텐츠 정책, 파일 완결성. 각각 PASS/WARN/FAIL 배지.

**결과별 액션:**

| 결과 | 의미 | 사용자 액션 |
|------|------|-----------|
| **PASS** | 모든 항목 통과 | "패키징 진행" 버튼 활성화 |
| **WARNING** | 경미한 문제 발견 | "무시하고 패키징" + "돌아가서 수정" 선택지 |
| **FAIL** | 심각한 문제 발견 | "돌아가서 수정" 버튼만 표시 + 어느 단계로 돌아갈지 선택 |

패키징 화면에서 산출물 트리 + 개별/전체 ZIP 다운로드.

## 5. 공통 컴포넌트 설계

### 5.1 스텝 진행 표시자 (`StepProgress`)

모든 Job 상세 화면 상단에 고정. 수평 스템퍼 7단계: **리서치 → 대본 → 이미지 → 영상 → TTS/합성 → QC → 패키징**

**색상:** 완료=`#27AE60`, 진행중=`#3498DB`, 대기=`#BDC3C7`, 에러=`#E74C3C`. 완료된 단계 클릭 시 해당 단계로 이동.

### 5.2 에셋 카드 (`AssetCard`)

이미지/영상용 공통 카드: 프리뷰(16:9) + 상태 배지 + 버전번호 + 액션 버튼(승인/재생성/복원/상세) + 씬번호/나레이션 요약.

### 5.3 버전 히스토리 (`VersionHistory`)

에셋 상세 모달에서 세로 타임라인으로 버전 변경 이력 표시. 두 버전 나란히 비교 + 복원 버튼.

### 5.4 실시간 로그 패널 (`ActivityLog`)

우측 패널. 타임스탬프 + 이벤트 아이콘 + 메시지. 색상코드 (system=회색, user=파랑, error=빨강, tool=보라). 전체/시스템/사용자/에러 필터 탭.

### 5.5 확인 모달 (`ConfirmModal`)

중요 액션 전 확인. 제목 + 설명 + 확인/취소. 위험 액션(QC 무시 등)은 빨간색 버튼.

## 6. 상태 머신 시각화

모든 Job은 `VALID_TRANSITIONS` 상태 머신에 의해 관리됩니다. UI는 이 상태를 실시간으로 반영합니다.

### 6.1 주요 상태 전이

| From | → | To | 트리거 |
|------|---|-----|--------|
| `searching` | → | `references_presented` | 검색 완료 |
| `references_presented` | → | `reference_selected` | 사용자 선택 |
| `script_pending_approval` | → | `script_approved` | 사용자 승인 |
| `images_pending_approval` | → | `images_fully_approved` | 전체 승인 |
| `images_fully_approved` | → | `video_generating` / `tts_generating` | 타입별 분기 |
| `compose_done` | → | `qc_reviewing` → `qc_passed` | 자동 QC |
| `qc_passed` | → | `exporting` → `exported` | 패키징 완료 |

### 6.2 UI 상태 표현 매핑

| 상태 그룹 | UI 표현 | 사용자 액션 |
|-----------|---------|-----------|
| `*_generating`, `*_syncing` | 로딩 스피너 + 진행률 | 대기 / 취소 |
| `*_pending_approval` | 승인 대기 배지 + 카드 활성화 | 승인 / 수정 / 재생성 |
| `qc_passed`, `exported` | 완료 배지 + 다운로드 활성화 | 패키징 / 다운로드 |
| `error`, `*_failed` | 빨간 배너 + 복구 옵션 | 재시도 / 복구 |

## 7. 기술 스택 & 구현 계획

### 7.1 프론트엔드

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| **프레임워크** | React (Vite) | CLAUDE.md 기준 기본 스택. 빠른 HMR |
| **언어** | JavaScript (JSX) | 소규모 프로젝트. 빠른 프로토타입 우선 |
| **상태 관리** | Zustand + React Query | UI 상태 + 서버 데이터 캐싱/폴링 |
| **스타일링** | Tailwind CSS | 유틸리티 기반 빠른 UI 구축 |
| **라우팅** | React Router v6 | SPA 라우팅, 중첩 라우트 지원 |
| **실시간 통신** | WebSocket (Socket.IO) | Job 상태, 진행률, 로그 실시간 수신 |

### 7.2 백엔드

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| **런타임** | Node.js + Express | 기존 TS 코드베이스 통일. 바이브코딩 스택 |
| **API** | REST + WebSocket | REST: CRUD. WS: 실시간 상태/로그 |
| **오케스트레이터** | 기존 `orchestrator.ts` | 상태 머신 + 이벤트 시스템 활용 |
| **외부 연동** | MCP 서버 5개 | youtube-data, comfyui, shell, filesystem, web-search |

### 7.3 구현 로드맵

| Phase | 범위 | 주요 작업 | 기간 |
|-------|------|----------|------|
| **P1** | 기반 + 대시보드 | 프로젝트 셋업, 라우팅, 레이아웃, Job 목록 API | 3~4일 |
| **P2** | Step 1~3 UI | 작업생성, 리서치, 대본편집기, WebSocket | 4~5일 |
| **P3** | Step 4~5 UI | 이미지 그리드, 버전관리, 영상프리뷰, TTS | 5~7일 |
| **P4** | Step 6~7 + 폴리싱 | QC 대시보드, 패키징, 통합테스트 | 3~4일 |

### 7.4 주요 API 엔드포인트

| Method | Endpoint | 설명 |
|--------|----------|------|
| **GET** | `/api/v1/jobs` | Job 목록 조회 |
| **POST** | `/api/v1/jobs` | 새 Job 생성 |
| **GET** | `/api/v1/jobs/:id` | Job 상세 조회 |
| **POST** | `/api/v1/jobs/:id/search` | YouTube 검색 실행 |
| **POST** | `/api/v1/jobs/:id/select-reference` | 레퍼런스 선택 |
| **POST** | `/api/v1/jobs/:id/approve-script` | 대본 승인/수정 |
| **POST** | `/api/v1/jobs/:id/scenes/:sid/regen` | 씬 이미지 재생성 |
| **POST** | `/api/v1/jobs/:id/approve-images` | 전체 이미지 승인 |
| **POST** | `/api/v1/jobs/:id/compose` | 영상+TTS+자막+BGM 합성 |
| **GET** | `/api/v1/jobs/:id/qc` | QC 검수 결과 |
| **POST** | `/api/v1/jobs/:id/export` | 패키징 + 다운로드 URL |

**WebSocket 이벤트:** `job:status_changed`, `job:progress_update`, `job:log_entry`, `job:asset_generated`, `job:error`
