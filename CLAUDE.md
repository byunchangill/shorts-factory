# AI Shorts Production System
# 리서치 기반 쇼츠 제작 파이프라인

## 프로젝트 목적

검색어 하나로 YouTube 인기 영상을 분석하고, 구조만 오마주하여
저작권 안전한 새 콘텐츠를 자동 생산하는 **반자동 파이프라인**.
단순 "쇼츠 생성기"가 아니라, 리서치 → 분석 → 재창작 → 생성 → 검수 → 패키징까지 이어지는 콘텐츠 제작 운영체제다.

## 3가지 콘텐츠 타입

| 타입 | 키워드 | 영상화 | 핵심 |
|------|--------|--------|------|
| 🐶 강아지 AI쇼츠 | 강아지, 개, 반려견 | ✅ 필요 | 감정선, 캐릭터 일관성, 표정 |
| 📖 썰툰 | 썰, 썰툰, 실화 | ❌ 이미지만 | 훅, 반전, 속도감, 대본 |
| 💊 건강 쇼츠 | 건강, 효능, 상식 | ✅ 필요 | 신뢰감, 가독성, 과장 금지 |

## 핵심 원칙 (모든 작업에 적용)

### 1. 일관성 최우선 (Consistency First)
이미지/영상 생성에서 일관성 > 참신함.
캐릭터 외형, 화풍, 팔레트, 프레이밍, 감정 톤을 씬 간 유지.
**반드시 Style Bible + Character Bible을 먼저 정의한 후 이미지 생성.**

### 2. 오마주 ≠ 복사 (Policy-Aware Creation)
원본 영상에서 추출하는 것: 구조, 감정 흐름, 훅 패턴, 페이싱
원본 영상에서 가져오지 않는 것: 문장, 대사, 캐릭터, 구체적 에피소드, 제목 패턴
**대본 생성 후 반드시 유사도 검증 수행.**

### 3. 씬 단위 재생성 (Controlled Regeneration)
전체를 다시 만들지 않는다.
마음에 안 드는 씬만 재생성하되, 나머지는 보존.
이전 버전 복원 가능하도록 버전 로그 유지.

### 4. 재사용 가능 설계 (Reusability First)
특정 채널/토픽에 종속되지 않는 공통 워크플로우.
Skills, 스키마, 정책을 다른 쇼츠 프로젝트에도 재활용 가능하게.

### 5. 추적 가능성 (Traceability)
모든 생성 단계에 기계가 읽을 수 있는 로그를 남긴다:
선택한 레퍼런스, 분석 요약, 대본 버전, 씬 계획, 프롬프트 버전, 재생성 이력, 최종 내보내기 로그.

### 6. 산출물 완결성 (Production-Ready Outputs)
완료된 작업은 반드시 다운로드 가능한 에셋 패키지를 생성:
최종영상, 개별이미지, 개별영상, 음성, 대본, 프롬프트, 메타데이터.

### 7. 사용 영상 배제 (Used Video Exclusion)
한번 오마주에 사용한 YouTube 영상은 이후 검색에서 **영구 배제**.
5개 후보에 올랐으나 미선택된 영상은 **7일간 배제**.
이를 통해 중복 제작을 방지하고 콘텐츠 다양성을 확보한다.
배제 이력은 `config/used_videos.json`에 저장.

### 8. 상태 추적 (State Machine)
모든 Job은 상태 머신으로 관리. 생성 상태와 **승인 상태**를 분리.
실패/부분완료/재수정/복원 흐름을 포함.
사용자가 승인했는지 여부가 상태에 명시적으로 들어간다.
상세 상태 전이는 `schemas/data-models.md` 참조.

### 9. 운영 안전 (Operational Safety)
- 사용자 입력을 셸 명령에 직접 삽입하지 않는다 (템플릿 방식)
- 허용 명령만 실행한다 (shell MCP --allow 리스트)
- 출력 경로는 화이트리스트로 제한한다
- 모든 실패에 로그를 남긴다
- 재시도는 씬당 최대 3회로 제한한다

## 워크플로우 (7단계)

```
[Step 1-2] 키워드 입력 → YouTube TOP 5 검색 → 구조 분석 → 사용자 선택
    ↓
[Step 3]   오마주 대본 생성 → 유사도 검증 → 사용자 확인/수정
    ↓
[Step 4]   Style Bible + Character Bible 정의 → 이미지 생성 → 개별 재생성
    ↓
[Step 5a]  영상화 (🐶강아지/💊건강만, 썰툰 제외) → 개별 재생성
    ↓
[Step 5b]  TTS 나레이션 → 음성-영상 싱크 → 자막+BGM 합성
    ↓
[Step 6]   최종 QC 검수
    ↓
[Step 7]   파일 패키징 → 다운로드
```

### 사용자 인터랙션 포인트
- Step 2 후: 5개 영상 분석 → **사용자가 1개 선택**
- Step 3 후: 대본 확인 → **수정 요청 가능**
- Step 4 후: 이미지 확인 → **개별 재생성 / 이전 버전 복원**
- Step 5a 후: 영상 확인 → **개별 재생성 가능**
- Step 6 후: QC 결과 → **수정 또는 통과**

## 콘텐츠 타입별 운영 규칙

### 🐶 강아지 AI쇼츠
- **우선순위**: 감정 훅 > 캐릭터 일관성 > 표정/포즈 > 영상 무브먼트 > 음성 톤
- **스타일**: 따뜻한 수채화 일러스트 / 소프트 애니메 스타일
- **주의**: 잔인한 설정 금지, 학대 묘사 과도 금지, 리얼리즘보다 감정 전달 우선

### 📖 썰툰
- **우선순위**: 훅 > 사건 구조 > 반전 > 가독성 > 컷 전환 속도
- **스타일**: 한국 웹툰 스타일, 클린 라인아트, 비비드 컬러
- **주의**: 실존 인물 특정 금지, 특정 집단 비방 금지, 자극만 세고 결말 약하면 이탈률 높음
- **특이**: 말풍선/텍스트를 이미지에 넣지 않음 → 자막으로 처리

### 💊 건강 쇼츠
- **우선순위**: 사실 안정성 > 가독성 > 훅 > 구조 > 텍스트 강조
- **스타일**: 클린 인포그래픽, 모던 플랫 일러스트, 실제 얼굴 X
- **컴플라이언스**: `schemas/data-models.md`의 HealthComplianceRules 참조
  - **주장 4등급 분류**: 안전(L1) → 조건부(L2) → 주의(L3) → 금지(L4)
  - **금지 표현**: "완치", "의사가 숨기는", "무조건", "기적", "100% 효과", "만병통치"
  - **출처 4등급**: 공인기관(T1) → 의료기관(T2) → 일반미디어(T3) → 불명(T4, 사용금지)
  - **자동 면책 삽입**: 나레이션 마지막 씬 + YouTube 설명란 하단
- **원칙**: Level 4 주장은 자동 재작성. Level 2~3은 조건부 표현 + 면책 필수.

## 이미지 생성 규칙 (최우선)

**씬 이미지 생성 전 반드시 아래 순서를 지킨다:**
1. Global Style Bible 정의 (화풍, 팔레트, 카메라, 렌더링)
2. Character Bible 정의 (캐릭터 외형, 의상, 핵심 특징)
3. Negative Prompt 정책 정의
4. 일관성 보존 규칙 확인
5. 그 다음에 씬별 이미지 생성

**재생성 시 기본 보존 항목:**
얼굴, 신체, 의상/털 패턴/핵심 외형, 렌더링 스타일, 팔레트, 종횡비

**재생성 시 변경 가능 항목:**
표정, 포즈, 구도, 손/발 디테일, 소품, 배경 디테일

## 대본 생성 규칙 (오마주 안전)

- 원문 문장 재사용 금지
- 원본 사건 순서 그대로 복제 금지
- 제목 패턴 너무 유사하게 금지
- 의역 표절 금지
- 추상적 성공 패턴(페이싱, 긴장감, 훅 스타일)만 보존
- 건강 콘텐츠: "도움이 될 수 있다", "일반적으로 알려진", "지원할 수 있다" 표현 사용

## 데이터 모델

모든 작업은 Job 단위로 관리. 상세 스키마는 `schemas/` 디렉토리 참조.
- `Job`: 작업 단위 (키워드, 상태, 타임스탬프)
- `ReferenceVideo`: 분석된 원본 영상
- `GeneratedScript`: 생성된 대본 + 유사도 검증 결과
- `Scene`: 씬별 정보 (나레이션, 비주얼, 타이밍)
- `ImageAsset`: 이미지 버전 관리 (시드, 프롬프트, 승인 상태)
- `VideoAsset`: 영상 버전 관리

## 기술 스택

| 용도 | 도구 | 비용 |
|------|------|------|
| YouTube 검색/분석 | YouTube Data API v3 | 무료 할당량 |
| 대본 생성 | Claude Code (구독 포함) | 추가 비용 없음 |
| 이미지 생성 | Stable Diffusion + IP-Adapter | 로컬 GPU |
| 영상화 | FFmpeg Ken Burns / AnimateDiff | 무료 / 로컬 GPU |
| TTS | Edge TTS | 무료 |
| 합성 | FFmpeg | 무료 |
| 유사도 검증 | Claude Code (직접 분석) | 추가 비용 없음 |
